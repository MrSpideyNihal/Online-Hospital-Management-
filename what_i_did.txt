  - Overview (page.tsx): Stats grid (6 metrics), quick actions (4 buttons),

  - Patients (patients/page.tsx): Search, add patient dialog (full form),

  - Appointments (appointments/page.tsx): List/today views, status filtering,

  - OPD Visits (visits/page.tsx): Live queue with status, quick check-in

  - Dental Chart (dental-chart/page.tsx): Interactive SVG with 32 teeth

  - Prescriptions (prescriptions/page.tsx): Prescriptions table with

  - Billing (billing/page.tsx): Revenue stats, invoices table with
    payment status (paid/pending/partial), export buttons
  - Reports (reports/page.tsx): Summary stats, chart placeholders for
    Recharts, top services performance bars
  - Settings (settings/page.tsx): 4 tabs (General, Branding with live
    color preview, Location with Google Maps embed, QR Code)

[✅] 5. HOSPITAL PUBLIC WEBSITE (src/app/hospitals/[slug]/*)
  - Homepage: Customizable hero (dynamic colors from hospital settings),
    services grid with pricing, doctors section with avatars, patient
    testimonials, booking CTA, footer
  - About page: Hospital story, contact info (address, phone, email, hours),
    Google Maps placeholder, doctor team section

[✅] 6. SUPER ADMIN PANEL (src/app/admin/*)
  - Layout: Red-themed sidebar, navigation, admin mode indicator
  - Dashboard: Platform-wide stats (hospitals, patients, revenue, growth),
    pending hospital approvals with approve/reject, expiring subscriptions
  - Hospitals: Hospital table with search, status filter (approved/pending/
    frozen/rejected), approve/reject/freeze actions, plan badges
  - Subscriptions: Revenue stats, subscription list with remind/extend actions
  - Analytics: Platform stats, hospitals by city distribution, revenue by plan
  - Notifications: Notification list with type badges, send/resend functionality

[✅] 7. DATABASE SCHEMA (supabase/schema.sql)
  Tables: hospitals, profiles, patients, doctors, appointments, visits,
  dental_charts, treatments, prescriptions, invoices, notifications,
  hospital_services, testimonials
  - Row Level Security (RLS) on all tables for multi-tenant isolation
  - Triggers for updated_at timestamps
  - Functions for patient_id_number and invoice_number generation
  - Indexes for performance
